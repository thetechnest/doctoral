Database Management Systems

This module focuses on database design, ER diagrams, normalization, and SQL querying. It covers query optimization, distributed and object-oriented databases, and an overview of commercial DBMS platforms.

Key Topics Explained

1. ER Diagrams and Data Models: Entity-Relationship (ER) diagrams represent data entities, relationships, and constraints. Data models include hierarchical, network, and relational models.

2. Relational Database Design: Organizes data into tables (relations) with keys and constraints for referential integrity.

3. Normalization: Reduces data redundancy. Includes 1NF (atomicity), 2NF (partial dependency removal), 3NF (transitive dependency removal), BCNF, and 4NF.

4. Limitations of Normalization: Excessive normalization can degrade query performance and increase join complexity.

5. SQL and QBE: Structured Query Language (SQL) is used for querying and data manipulation; Query By Example (QBE) allows visual query creation.

6. Query Processing and Optimization: Includes parsing, translation, optimization, and execution to improve efficiency.

7. Centralized and Distributed Databases: Centralized DB stores all data in one site; distributed DB splits data across multiple locations, requiring synchronization.

8. Database Security: Uses authorization, encryption, and auditing to protect sensitive data.

9. Object-Oriented DBMS (OODBMS): Combines database capabilities with object-oriented programming, storing data as objects rather than tables.

10. Commercial DBMS Examples: Oracle, MySQL, PostgreSQL, MS SQL Server, and MongoDB are widely used enterprise systems.
